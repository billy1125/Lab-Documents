# 研究記錄說明

> 聲明：本規範會隨實驗室運作、研究需求與研究領域差異進行調整

## 一、先說清楚

本實驗室的研究可能包含：

- 人工智慧與工程實作
- 管理研究與商業分析
- 社會科學研究與資料分析

雖然研究方法不同，但有些基本要求是一樣的：

> **研究必須可理解、可追蹤、可重現、可交接。**

本系統的目的，不是增加形式上的負擔，而是避免以下問題：

- 做過的研究沒有留下可用紀錄
- 學生自己過一段時間也看不懂
- 論文或報告撰寫時無法回頭整理
- 學生畢業後資料、程式或分析流程中斷
- 不同研究類型之間沒有共同管理標準

## 二、核心原則

### 1. 沒有被記錄的研究，等同於沒有完成

研究不只看結果，也看過程是否能被說清楚。

至少要能回答：

- 你做了什麼？
- 為什麼這樣做？
- 你用了哪些資料、方法或工具？
- 得到了什麼結果？
- 下一步要做什麼？

### 2. 研究應能被他人理解

你的紀錄不應只讓你自己看得懂，而應讓：

- 指導老師
- 未來接手的人
- 幾個月後的你自己

都能快速理解研究脈絡。

### 3. 研究應能被追蹤

所有研究結果，都應能回溯到：

- 對應的研究問題
- 使用的資料
- 使用的方法
- 使用的版本
- 分析或實作過程

### 4. 研究應能被延續

研究不是一次性的嘗試，而是可被累積的工作。

這代表：

- 檔案不能亂放
- 資料不能只存在個人電腦
- 程式碼、分析、文件需要有基本結構
- 畢業或交接時，內容應可延續

## 三、適用範圍

本系統適用於以下類型研究：

### A. AI / 工程研究
例如：

- 機器學習
- 深度學習
- 系統開發
- 模型訓練與評估
- 演算法實作

### B. 管理研究
例如：

- 管理議題分析
- 商業模式研究
- 個案研究
- 問卷資料分析
- 組織或策略研究

### C. 社會科學研究
例如：

- 問卷調查
- 訪談研究
- 行為研究
- 次級資料分析
- 實證研究

## 四、統一要求

無論研究類型為何，每位成員都必須完成以下內容。

### 1. 建立研究紀錄

每個研究主題都應有持續更新的紀錄。

紀錄形式可以是：

- Markdown
- Google Docs
- Notion
- 其他經同意的數位格式

但原則上應符合以下要求：

- 可存取
- 可保存
- 可搜尋
- 可交接

### 2. 每次研究活動都需留下紀錄

每一次重要的研究活動，至少需記錄：

- 日期
- 負責人
- 研究問題或目標
- 使用的方法
- 使用的資料或材料
- 執行過程
- 結果
- 問題與觀察
- 下一步

### 3. 不只記錄成功，也記錄失敗

請不要只記錄看起來漂亮的結果。

以下內容同樣需要記錄：

- 失敗的嘗試
- 無效的方法
- 不成立的假設
- 中途修正的原因

這些內容對後續研究同樣重要。

## 五、研究資料與文件管理

### 1. 不得只存於個人設備

以下情況不被接受：

- 研究資料只在個人電腦
- 程式碼未同步
- 文件只有私人帳號可存取
- 離開實驗室後他人無法接手

所有正式研究資料，應存放於實驗室指定或可共享的空間。

### 2. 檔案命名需清楚

請避免以下命名方式：

- `test`
- `final`
- `final2`
- `new`
- `最新版`
- `真的最終版`

建議命名包含：

- 日期
- 主題
- 版本
- 內容類型

例如：

- `2026-04-16-literature-review.md`
- `survey-analysis-v01.xlsx`
- `model-baseline-exp03.md`

### 3. 資料夾結構需一致

每個研究主題建議至少包含：

- `docs/`：研究文件、會議紀錄、文獻整理
- `data/`：資料
- `analysis/`：分析過程或結果
- `code/`：程式碼（如適用）
- `outputs/`：圖表、模型、表格、報告輸出

## 六、依研究類型補充要求

### A. AI / 工程研究補充要求

若研究涉及程式、模型或系統，需額外做到：

- 使用 Git 或等效版本控制
- 記錄程式版本
- 記錄資料版本
- 記錄主要參數設定
- 讓結果能對應到特定版本與設定

至少需能回答：

- 哪個版本產生這個結果？
- 使用哪一份資料？
- 參數是什麼？
- 是否可再次執行？

### B. 管理研究補充要求

若研究涉及個案、商業分析或管理理論，需額外做到：

- 清楚說明研究問題
- 清楚記錄理論依據或文獻依據
- 清楚說明資料來源
- 清楚記錄分析邏輯
- 區分「資料事實」與「研究者推論」

至少需能回答：

- 這個結論的依據是什麼？
- 使用哪些資料？
- 為什麼採用這種分析方法？
- 是否有其他可能解釋？

### C. 社會科學研究補充要求

若研究涉及問卷、訪談、受試者資料或社會資料分析，需額外做到：

- 清楚記錄研究設計
- 清楚記錄樣本來源與篩選條件
- 清楚記錄資料收集方式
- 清楚記錄資料清理與轉換過程
- 符合研究倫理與隱私要求

至少需能回答：

- 資料如何取得？
- 樣本如何選擇？
- 是否有資料清理或排除標準？
- 是否涉及匿名與個資保護？

## 七、研究紀錄最低標準

每份研究紀錄至少應包含以下欄位：

1. 基本資訊  
2. 研究問題或目標  
3. 方法或研究設計  
4. 使用資料 / 材料 / 樣本  
5. 執行過程  
6. 結果  
7. 問題與觀察  
8. 結論  
9. 下一步

請使用實驗室提供的統一模板。

## 八、交接與畢業前整理

在研究結束、學生畢業或交接前，需完成以下整理：

- 研究主題摘要
- 主要資料位置
- 主要文件位置
- 程式碼或分析檔案整理
- 研究結果說明
- 尚未完成事項
- 建議後續方向

交接的最低標準是：

> **後續接手的人，可以在合理時間內理解並延續你的研究。**

## 九、常見錯誤觀念

### ❌「我自己知道就好」
→ 研究不是只給現在的自己看

### ❌「成功再記錄就好」
→ 失敗與修正同樣重要

### ❌「先做，之後再整理」
→ 多數情況下，之後不會完整整理

### ❌「不同領域不用統一管理」
→ 研究方法不同，但管理邏輯需要一致

## 十、最後的重點

請記住以下四件事：

1. **沒有紀錄，就沒有完成研究**
2. **研究必須能被理解**
3. **研究必須能被追蹤**
4. **研究必須能被延續**

本系統的目的不是增加形式負擔，而是：

> **讓研究不會因為人離開、時間經過或檔案混亂而中斷。**
